# GitHub Runner End-to-End Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `spook create runner-01 --github-runner --github-repo org/repo --github-token-keychain <acct>` produces, with zero human interaction inside the guest, a VM whose GitHub Actions runner registers and picks up a real job.

**Architecture:** Reuse three existing mechanisms, add zero new privilege paths: (1) the host seeds `Spooktacular Provisioner.pkg` into the VM bundle's `provision/` directory, which every macOS boot already attaches as the virtio-fs share `spook-provision` (`VirtualMachineConfiguration.applyProvisioning`); (2) `SetupAutomation` — which already types `sudo systemsetup -setremotelogin on` into Terminal as admin — gains four more typed commands that mount the share and run `sudo installer -pkg`, installing the provisioner LaunchDaemon root:wheel via Apple's own installer, eliminating the manual pkg-wizard step; (3) the orphaned `GitHubRunnerService.issueRegistrationToken` is wired into the create flow so the Keychain holds a long-lived **PAT** and a fresh 1-hour registration token is minted seconds before injection + boot (fixing the token-TTL race). The runner template is rewritten to run under the root provisioner but configure/execute the runner as `admin` via a `UserName`-scoped LaunchDaemon (survives reboots, no login session needed, runner not root, non-blocking so the provisioner's trigger-archival loop completes).

**Tech Stack:** Swift 6 / SwiftPM, Virtualization.framework (`VZVirtioFileSystemDeviceConfiguration` share already wired), GitHub REST (`POST /repos/{o}/{r}/actions/runners/registration-token`), launchd, `installer(8)`, `gh` CLI for the live test.

```bash
#!/bin/bash
# GitHub Actions runner bootstrap — executed as root by the Spooktacular
# provisioner LaunchDaemon on first boot. Configures the runner as the
# admin user and hands it to launchd, then exits (network wait below).
set -euo pipefail
REPO='<esc repo>'; TOKEN='<esc token>'; RUNNER_NAME='<esc name-or-empty>'
LABELS='<esc comma-joined-or-empty>'; EPHEMERAL=<0|1>
RUNNER_USER="admin"; RUNNER_DIR="/Users/${RUNNER_USER}/actions-runner"
# network wait: Setup Assistant is done but DHCP may lag on first boot
for _ in $(seq 1 60); do curl -fsS https://api.github.com >/dev/null 2>&1 && break; sleep 2; done
sudo -u "$RUNNER_USER" mkdir -p "$RUNNER_DIR"
TARBALL_URL=$(curl -fsSL https://api.github.com/repos/actions/runner/releases/latest \
  | /usr/bin/python3 -c 'import json,sys;print(next(a["browser_download_url"] for a in json.load(sys.stdin)["assets"] if "osx-arm64" in a["name"] and a["name"].endswith(".tar.gz")))')
sudo -u "$RUNNER_USER" bash -c "cd '$RUNNER_DIR' && curl -fsSL -o runner.tar.gz '$TARBALL_URL' && tar xzf runner.tar.gz && rm runner.tar.gz"
CONFIG="./config.sh --url 'https://github.com/$REPO' --token '$TOKEN' --unattended --replace"
[ -n "$RUNNER_NAME" ] && CONFIG="$CONFIG --name '$RUNNER_NAME'"
[ -n "$LABELS" ] && CONFIG="$CONFIG --labels '$LABELS'"
[ "$EPHEMERAL" = "1" ] && CONFIG="$CONFIG --ephemeral"
sudo -u "$RUNNER_USER" bash -c "cd '$RUNNER_DIR' && $CONFIG"
cat > /Library/LaunchDaemons/com.spooktacular.github-runner.plist <<'PLIST'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0"><dict>
    <key>Label</key><string>com.spooktacular.github-runner</string>
    <key>UserName</key><string>admin</string>
    <key>WorkingDirectory</key><string>/Users/admin/actions-runner</string>
    <key>ProgramArguments</key><array><string>/Users/admin/actions-runner/run.sh</string></array>
    <key>RunAtLoad</key><true/>
    <key>KeepAlive</key>
    <KEEPALIVE/>
    <key>StandardOutPath</key><string>/var/log/spooktacular-runner.log</string>
    <key>StandardErrorPath</key><string>/var/log/spooktacular-runner.err.log</string>
</dict></plist>
PLIST
# substitute <KEEPALIVE/> with <true/> (persistent) or <false/> (ephemeral) at template-render time, not in bash
chown root:wheel /Library/LaunchDaemons/com.spooktacular.github-runner.plist
chmod 644 /Library/LaunchDaemons/com.spooktacular.github-runner.plist
launchctl bootstrap system /Library/LaunchDaemons/com.spooktacular.github-runner.plist || true
```
