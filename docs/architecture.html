Clean architecture, Apple-first

One domain model. Four front-ends.

Every control surface — the SwiftUI app, the spook CLI, the mTLS HTTP server, the Kubernetes operator — calls into the same typed use cases. Layer boundaries are enforced at build time; nothing in SpookCore can import AppKit.

Data flow

One bundle format, three ways to drive it.

VirtualMachineBundle is the on-disk source of truth. Every front-end reads and writes the same directory layout.

On-disk layout

~/.spooktacular/vms/macos-26-base.vm 
.
├── config.json          # spec (CPU, RAM, disk, net)
├── metadata.json        # bundle id, timestamps, icon
├── disk.img             # APFS sparse disk image
├── hardware-model.bin   # VZ hardware fingerprint
├── machine-identifier.bin # unique per bundle
├── auxiliary.img        # VZ aux storage
└── snapshots/
    └── golden/           # COW-linked disk

Atomic writes & typed errors

writeSpec(_:to:) writes to a sibling tmp file, fsyncs, then atomically renames. A crashed process can't leave a half-written config. Every boundary returns a typed error — no silent nil fallbacks.

  • VirtualMachineBundle.create(at:spec:)
  • .load(from:) / .writeSpec(_:to:)
  • .writeMetadata(_:to:)
  • SnapshotManager.save / restore / list / delete
  • CloneManager.clone(source:to:) (APFS COW)
  • DiskInjector.inject(script:into:) (LaunchDaemon)
Deploy how you like

Three topologies. Pick the one that matches your infrastructure.

Laptop / workstation

Download the SwiftUI app from the Mac App Store. The bundled spook binary lives in /Applications/Spooktacular.app/Contents/MacOS/. No server, no cluster, no config.

App Store · single host

Dedicated Mac host

A rack-mounted Mac mini (or a leased dedicated Mac in a datacenter). Install spook serve under LaunchDaemon. Mint an mTLS cert from your internal CA. The HTTP API exposes the full control surface.

LaunchDaemon · mTLS

Kubernetes fleet

Install the Helm chart. CRDs register. Each node runs a spook-controller DaemonSet that reconciles MacOSVM and RunnerPool resources. Validating webhook enforces tenant isolation + the 2-per-host cap.

Helm · operator · webhook
Provisioning checklist is shipped as code. spook doctor --strict walks the 18-item DEPLOYMENT_HARDENING.md preflight: TLS, mTLS, TLS-1.3 floor, API bearer in Keychain, guest-agent tokens, RBAC active, federated IdP, JWKS pinned, audit JSONL writable, append-only audit, signing key perms, distributed lock, tenancy mode, insecure mode OFF, hardened runtime, code-signing timestamp, etc. Non-zero exit means the deployment is not hardened.
HTTP API

A REST control plane, not a toy server.

Built on Network.framework, not URLSession. Hardened against the usual boundary attacks out of the box.

Request hygiene

Body size enforced before trusting Content-Length. Duplicate Content-Length rejected per RFC 7230 §3.3.2. Slow-loris defense tracks elapsed time from first byte. X-Forwarded-For only honored when operator opts in via SPOOK_TRUST_FORWARDED_FOR=1.

Errors as a contract

{error: {code, message, requestId}} on every failure. 12 typed error codes. X-Request-ID minted per response — the audit record, the trace span, and the error envelope all reference the same ID.

Rate limiting

Split read/write buckets — a scraper hitting GET /vms can't burn the write quota. Per-token and per-IP limits combined. 429 with Retry-After.

Auth & authz

Bearer token for reads, mTLS for writes (operator choice — both can be required). RBAC on every handler; the authorization decision goes into the audit stream.

Kubernetes

A real operator. Typed conditions. Drain on terminate.

Two CRDs

MacOSVM declares one VM on a specific host or a label-selected pool. RunnerPool declares a min/max pool of runners against a repo or org. Both track observedGeneration and emit typed conditions: Available, Progressing, Degraded, PoolReady, ScaleUpInProgress, CapacityExhausted.

  • Validating webhook blocks tenant/namespace violations at admission
  • 2-VMs-per-host cap enforced both at admission and at reconcile
  • Watch-stream reconnect with exponential backoff (5→10→30s)
  • watch_stream_reconnect_total metric tracks recovery
  • Helm chart ships NetworkPolicy, PDB, ServiceMonitor, PrometheusRule

Observability

SLOs as code. Dashboards as JSON.

The repo ships an SLO catalog (docs/observability/slo-catalog.md) with 10 user-facing + platform SLIs. Each one has a PromQL expression, target, evaluation window, and an alert rule name you can kubectl apply.

  • OpenTelemetry trace export (OTLP HTTP/JSON) on every API request
  • Prometheus metrics with a typed, bounded-cardinality registry
  • Two Grafana dashboards (controller + fleet) as JSON
  • SLO-burn-rate alerts in PrometheusRule
  • Prometheus scrape config for K8s SD + EC2 SD with mTLS
  • Request-ID correlation across HTTP, audit, traces
  • Menu-bar status icon animates while any VM is transitioning
