Threat model

What's in scope. What isn't.

Designed against a specific threat model: malware running as the logged-in user on the operator's workstation. That moves a lot of controls — env vars, CLI flags, plaintext files — out of the acceptable design space.

In scope

A compromised process under the logged-in UID. A compromised CI runner attempting to exfiltrate the host's Mac fleet tokens. A compromised VM attempting lateral movement. Cross-tenant data leakage through shared Kubernetes state.

Out of scope

Root exploits on the host kernel (that's Apple's). Attackers with physical possession and unlock time. Side-channel attacks on the Secure Enclave. Apple Silicon firmware flaws.

Pre-1.0 no-compat-hedges rule. Legacy paths that accepted secrets via env vars, CLI flags, or files on disk were removed before 1.0 shipped — not gated behind a deprecation warning. The Keychain is the only accepted path. A feature that would weaken the threat model doesn't exist, not even as a compatibility shim.
Identity & authentication

Federated, pinned, replay-proof.

OIDC

Full JWT validation: signature, iss, aud, exp, nbf, iat, replay. JWKS pinned per issuer — no "discover and trust" default. iss validated before JWKS key selection — the IdP confusion vector is closed at the type level.

SAML

XML canon caps entity expansion + element depth (billion-laughs defense). CDATA fidelity preserved. Conditions validation fails closed — a missing NotBefore or NotOnOrAfter never silently skips time validation. JWKS for signing keys; pinned per IdP.

mTLS

TLS 1.3 floor. KeychainTLSProvider verifies cert/key pairs by signing a nonce and validating — an imported mismatched identity is rejected at load time, not silently broken at runtime. Cert pinning on the refinement protocol makes "turn off pinning" unreachable by construction.

Per-action MFA

Admin actions require Keychain ACL + LocalAuthentication per call, not a session token that outlives the user's intent. Satisfies OWASP ASVS V2.10 at the action boundary — an attacker with a stolen session token can't invoke admin APIs without re-authentication.

Secrets

Keychain-only, Secure-Enclave where possible.

Tokens, registration tokens, webhook secrets, audit signing keys — all resolved through a single path per type. Env, flag, and file paths were removed pre-1.0.

  • GitHub runner tokens — Keychain service com.spooktacular.github, account name per scope
  • Webhook signing secrets — Keychain service com.spooktacular.webhook
  • API bearer tokens — Keychain with env-var fallback only at startup
  • Guest-agent tokens — Keychain, scoped per channel
  • Audit signing keys — Keychain, 0600 perms enforced
  • Break-glass signing keys — Secure-Enclave-bound (P-256 in SEP)
  • Workload JWT signing — JWKS kid rotation with 24 h overlap
  • Host-side certs — Keychain with ACL + kSecAttrAccessibleWhenUnlockedThisDeviceOnly
No silent fallbacks. Every resolver throws a typed error with an actionable recoverySuggestion that tells the operator exactly which security add-generic-password command populates the missing item. No guessing, no chain-of-fallbacks hiding a misconfiguration.
Break-glass

Time-limited, single-use, signed, audited.

When a human needs to override RBAC — incident response, customer support — the mechanism is an explicit ticket, not a "sudo for the fleet" mode.

BreakGlassTicket

Ed25519-signed value type (RFC 8037). Compact bgt:<base64url-payload>.<base64url-sig> wire format with no alg header — eliminates JWT's algorithm-confusion surface rather than guarding against it.

Ticket payload: issuer, tenant, fleet, JTI, iat, exp, allowed scopes, max-uses. The signer key lives in the Secure Enclave; the verifier public key is pinned per issuer.

UsedTicketCache

Distributed JTI denylist backed by DynamoDB in production (DynamoDBFleetSingleton) or a local file in single-host deployments. Enforces single-use per JTI and max-uses across the fleet — an attacker can't replay a ticket on a different controller.

  • Four-gate server-side enforcement: port-tier, credential-path, credential check, handler-tier re-assertion
  • spook break-glass keygen + issue CLI
  • Agent loads verifier via SPOOK_BREAKGLASS_PUBLIC_KEY / _ISSUERS / _TENANT env
  • Every break-glass use emits an audit record with a separate severity class
Audit

RFC 6962 Merkle tree. Verifiable on a laptop.

Append-only JSONL with fsync on every write. Daily size-threshold rollover. S3 Object Lock dual sink with HEAD-verify after every PUT — if the object isn't under Object Lock, the write fails, loudly.

Shape

One record per decision, per state transition, per security event. Hash-chained: each record's hash includes the previous record's hash. Tree-head signed with Ed25519 every N records. HashChainAuditSink rehydrates on startup and throws on chain break.

Verify

spook audit verify --record <r> --audit-path <p> --tree-head <t> --public-key <k> reconstructs the Merkle inclusion proof and validates the signature. Supports P-256 ECDSA (current signer) and Ed25519 (CT-compatible legacy).

Dual sink

Writes go to the local JSONL and S3 Object Lock. A divergence alarm fires if the two diverge — an attacker can't silently drop records locally without the remote copy disagreeing.

SIEM

HMAC-signed webhook forwarder with exponential backoff and a bounded retry queue. Metric on loss so an outage is visible, not silent.

Network

Egress control, compiled.

  • Per-VM pf(8) rules generated from a typed policy — every outbound connection goes through the firewall
  • Isolated network mode — no network interface at all, host ↔ guest over VirtIO socket only
  • NAT — shared internet via host, separate MAC per VM, DHCP lease recovery via ARP
  • Bridged — VM gets its own IP on your LAN (requires the VM-networking entitlement)
  • HTTP body-size caps — checked before trusting Content-Length
  • Slow-loris defense — tracks elapsed time from the first byte of a request
  • Duplicate Content-Length rejected per RFC 7230 §3.3.2
  • X-Forwarded-For trust is operator-opt-in, not default
Data at rest

Encrypted on power-off. Verified on load.

Bundle protection

BundleProtection.apply sets FileProtectionType.completeUntilFirstUserAuthentication on portable Macs. The bundle — disk image, config, auxiliary storage — is encrypted at rest when the laptop is powered off, even if the FileVault recovery key is later compromised.

Desktops and rack-mounted Mac hosts get .none so pre-login LaunchDaemons can still read bundles at boot. The policy is automated via BundleProtection.recommendedPolicy() and overridable via SPOOK_BUNDLE_PROTECTION.

Integrity checks

CloneManager verifies machine-identifier.bin SHA-256 after write — a corrupt or truncated clone surfaces at clone time, not on next boot. SnapshotManager uses a staging directory renamed on success so a crashed save can't leave a half-written snapshot.

Compliance

Posture, not certification.

Spooktacular is not SOC 2 or ISO 27001 certified — certification applies to your deployment, not to an open-source codebase. What we ship is a posture: the controls, mapped 1:1 to the relevant control framework, auditable in git blame.
