Features

An in-depth survey of every capability in Spooktacular.

VM Lifecycle Management

Create from IPSW

Download Apple's official macOS restore images and install them into a new VM. The IPSW is cached locally by SHA256 so subsequent VMs skip the download. Spooktacular checks version compatibility before downloading — if the latest IPSW is newer than your host macOS, you get a clear error with actionable guidance instead of a wasted 13 GB download.

spook create my-vm --from-ipsw latest --cpu 8 --memory 16 --disk 100

Instant Cloning

APFS copy-on-write cloning duplicates a 30 GB VM in milliseconds. Both VMs share physical disk blocks — space is only consumed when one side writes. Every clone gets a fresh VZMacMachineIdentifier to prevent the Virtualization framework's most documented footgun.

spook clone base runner-01    # ~50ms for a 30 GB VM

Start, Stop, Pause

Boot VMs with a display window or headless for CI. Boot into macOS Recovery mode for system repair. Save and restore VM state (macOS 14+ snapshots).

spook start my-vm                    # display window
spook start my-vm --headless          # CI/server mode
spook start my-vm --recovery          # macOS Recovery
spook snapshot my-vm checkpoint-1     # save state
spook restore my-vm checkpoint-1      # restore state

Configuration Management

View and modify VM specs after creation. Machine-readable JSON output for automation pipelines.

spook get my-vm                # styled table
spook get my-vm --json          # JSON for automation
spook get my-vm --field cpu     # single value: "8"
spook set my-vm --cpu 12 --memory 32

Networking

NAT (Default)

Guest reaches the internet through the host. Host reaches the guest via DHCP-assigned IP. No entitlement needed. Works in the Mac App Store build.

Bridged

Guest gets its own IP on the host's LAN. Full bidirectional connectivity. Requires com.apple.vm.networking entitlement (Developer ID build only).

Isolated

No network interface attached. Zero connectivity. Use for secure builds where network exfiltration must be impossible.

Host-Only

Guest communicates with the host and other VMs but not the internet. Perfect for federated ML training where VMs need to coordinate without external access.

Provisioning (User Data)

Four strategies for running scripts on first boot, consistent across CLI, GUI, and Kubernetes:

Disk Inject zero-touch

Mounts the guest disk before boot and injects a standard macOS LaunchDaemon. The script runs automatically when macOS starts. No SSH, no agent, no network needed. The only mode that works on a completely vanilla macOS from IPSW.

SSH

Waits for the VM to boot, discovers its IP, connects via SSH, and runs the script. Real-time output streaming. Requires Remote Login enabled in the guest.

Guest Agent fastest

A lightweight agent inside the guest communicates over VirtIO socket (vsock). No networking required. Pre-installed in all Spooktacular OCI images.

Shared Folder

Script delivered via VirtIO shared folder. A watcher daemon in the guest executes new scripts automatically. No networking, no disk modification.

Shared Folders

Mount host directories into the guest at /Volumes/My Shared Files/ via VirtIO file system. Read-only or read-write per folder. Essential for:

  • CI build artifacts
  • ML training datasets (mount terabytes of data without copying into the VM)
  • Configuration files and scripts
  • Test fixtures and test data
spook share my-vm add ~/Projects --tag projects
spook share my-vm add /data/models --tag models --read-only
spook share my-vm list

Image Library

Cache IPSW and OCI images locally. Browse available images in the GUI sidebar. Add images by local file path or OCI registry reference.

spook pull ghcr.io/spooktacular/macos-xcode:15.4-16.2 base
spook create runner --clone-from base

Kubernetes Integration

Full Kubernetes-native VM management via custom resources:

  • MacOSVM CRD — single VM with full spec
  • MacOSVMPool CRD — managed pool of ephemeral runners with auto-replacement
  • Operator — Swift binary, watches resources, schedules onto Mac hosts
  • Capacity-aware scheduler — respects 2-VM kernel limit per host
  • Helm chart for installation

EC2 Mac Fleet Management

One-liner setup on AWS EC2 Mac dedicated hosts. Three fleet scaling models:

  • AWS Auto Scaling Groups — ASG manages host count, Spooktacular manages VMs
  • Kubernetes cluster-autoscaler — operator signals when capacity is needed
  • Built-in fleet controller — spook fleet talks to AWS API directly
curl -sSL https://spooktacular.dev/ec2-setup.sh | bash -s -- \
    --github-repo myorg/myrepo --github-token ghp_xxx --xcode 16.2
