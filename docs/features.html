01 · VM lifecycle

Create, clone, snapshot, restore.

Every primitive Apple's Virtualization.framework exposes — wrapped in a durable on-disk bundle format with atomic config writes, fsync'd snapshots, and regenerated machine identifiers on clone.

Shipped

Create from IPSW

Downloads the latest macOS restore image from Apple's signing service with resumable HTTP Range + If-Range and SHA-256 content-addressed caching. Or pass a local IPSW to skip the 10–20 GB fetch.

Shipped

APFS copy-on-write clone

100 GB clones in ~48 ms. Only bytes the clone later writes consume host storage. Each clone gets a fresh VZMacMachineIdentifier so both VMs run concurrently.

Shipped

Snapshots

Disk-level save / restore / list / delete. Atomic via staging directory renamed on success.

Shipped

Graceful + hard stop

stopGracefully(timeout:) sends VZ shutdown, falls back after N seconds; stopImmediately() pulls the cord. Callers opt in explicitly.

Shipped

Pause / resume / save / restore

Full VZ state lifecycle. VM state JSON tracks every transition; PIDFile identifies the owning process; transitions emit audit events.

Shipped

Setup Assistant automation

Unattended first-boot for macOS 15 (Sequoia) and 26 (Tahoe). Keyboard driver + screen reader pairing walks through every prompt, then waits for SSH to confirm completion.

Shipped

Hardware editor (stopped VMs)

Re-tune CPU, memory, disk, audio, microphone, and shared folders on any stopped VM. Disk can only grow — shrinking is rejected.

Shipped

2-VMs-per-host cap

Apple's EULA ceiling, enforced at the GUI, CLI, HTTP API, and the Kubernetes admission webhook. Multi-tenant quotas account for pending allocations.

Shipped

Data-at-rest protection

BundleProtection.apply sets FileProtectionType.completeUntilFirstUserAuthentication on portable Macs so the bundle is encrypted at rest after power-off.

02 · Provisioning

Four ways to bootstrap a guest. Pick what the network allows.

All four modes run the same script; they differ only in delivery. Templates for GitHub runners, OpenClaw agents, and Screen Sharing generate the script for you.

Shipped

disk-inject

Mounts the VM's APFS data volume via hdiutil before first boot, drops the script + a LaunchDaemon plist, unmounts. Works on a vanilla install with zero network. The default.

Shipped

ssh

Boots the VM, waits for SSH, scps the script, executes, captures exit code. Key path defaults to ~/.ssh/id_ed25519; user defaults to admin.

Shipped

agent (VirtIO socket)

Delivers via the Spooktacular guest agent over vsock. No network required, no SSH host-key churn, fastest path. Agent pre-installed in the published OCI image.

Shipped

shared-folder

Drops the script into a VirtIO FS share; a watcher daemon inside the base image picks it up and runs it. Works offline, survives first-boot races.

Built-in templates

v1.0.1

GitHub Actions runner

Downloads the runner, configures with your repo + registration token (read from the Keychain — env/flag/file paths removed pre-1.0), registers with the self-hosted/macOS/ARM64 label set. Ephemeral, warm-pool, or persistent mode.

v1.0.1

OpenClaw agent

Installs Node.js + the OpenClaw gateway so the VM becomes a sandboxed agent host. API keys arrive via a VirtIO shared folder — no secrets in the provisioning script.

v1.0.1

Remote Desktop

Enables macOS Screen Sharing on first boot and surfaces the VNC URL in the system log so a VNC client + spook ip is all you need.

03 · Guest agent

One VirtIO socket, three scope-separated channels.

The agent runs inside the guest on a LaunchDaemon. Three listening ports, each a different trust tier: read-only for health/observability, runner for what CI needs, break-glass for emergency admin gated by a Secure-Enclave-signed ticket.

read-only

Health & observability

Process list, listening ports, uptime, version, disk usage. Safe to expose to any host-side observer — no mutations, no secrets.

runner

CI operations

Exec, copy files in/out, open apps, fill clipboard. The port CI runners talk to for the workloads they actually need to do.

break-glass

Emergency admin

Ed25519-signed tickets required per call (RFC 8037, bgt:<payload>.<sig>). UsedTicketCache enforces single-use via a distributed JTI denylist.

Hardening. The agent HTTP server raises the vsock backlog to 64, enforces a 128-connection ceiling with 503 on overflow, sets SO_RCVTIMEO=5s on every accepted socket, and runs under a LaunchDaemon with ThrottleInterval=10 + SoftResourceLimits (256 file descriptors, 512 MiB memory). GuestAgentClient.request(timeout:) on the host side races the read against a deadline via TaskGroup.
04 · Runner pools

Ephemeral, warm, or persistent — pick by workload.

RunnerPool is a typed Kubernetes CRD and a first-class CLI primitive. The operator handles provisioning, scaling, draining, and cleanup. Every state transition emits an audit record.

Ephemeral

VM runs one job, deregisters from GitHub, gets destroyed, a fresh clone takes its place. The strictest isolation — every job on a byte-for-byte fresh macOS.

Warm pool

VMs recycled via a 10-check scrub validator that proves the guest is clean: no runner processes, no /Users/runner/work, no clipboard, no /tmp, no rogue LaunchAgents, no unexpected TCP listeners (SSH only).

Persistent

Long-lived VMs for workloads that can't tolerate cold-start (LLM sidecars, build caches). Operator still handles drain-on-shutdown.

Lifecycle guarantees

  • Drain-before-delete — notifies GitHub, polls for in-flight jobs with a deadline, then deletes.
  • IssuedTokenLedger tracks registration tokens, expires them at GitHub's 1-hour TTL.
  • Workload token issuer with real JWKS kid rotation, 24 h overlap window.
  • Per-label shell escape in the runner template closes a command-injection vector.
  • Keychain-only token resolution — env vars, flags, files were removed pre-1.0.
  • PoolAction.drainRunner(name, deadline:) — explicit, testable drain step.

06 · Security

Built for a Fortune-20 vendor review.

Every control is documented, tested, and cite-able. Full posture on the security page; headlines below.

mTLS everywhere

Keychain-resident certs. KeychainTLSProvider verifies cert/key pairs by signing a nonce before trust — mismatched identities are rejected, not silently broken. TLS 1.3 floor enforced.

Federated identity

OIDC and SAML with JWKS pinning, strict aud / iss / replay validation. iss validated before JWKS key selection (cross-IdP confusion fix). SAML verifier fails closed on missing Conditions. XML canon caps entity expansion + element depth (billion-laughs defense).

RBAC + audit

Multi-tenant RBAC with an audit record on every decision. AuthzOutcome distinguishes deny from transient error. Append-only JSONL with fsync on every append; S3 Object Lock dual sink with HEAD-verify.

Break-glass tickets

Ed25519-signed (RFC 8037). Compact bgt:<payload>.<sig> wire format with no alg header — eliminates JWT's algorithm-confusion surface. UsedTicketCache is distributed; per-JTI and per-fleet single-use.

Secure Enclave signing

Break-glass verifier keys are SEP-bound via SecureEnclave.P256.Signing.PrivateKey. Signed-request key material held under Keychain ACL + LocalAuthentication per-action — satisfies ASVS MFA V2.10 at the action boundary.

Audit verify

spook audit verify reconstructs RFC 6962 Merkle inclusion proofs from the audit store and validates signatures against a pinned P-256 or Ed25519 key (CT compatibility).

Egress control

Per-VM pf(8) rules compiled from a typed policy. Blocks outbound connections not on the allowlist before they leave the host.

Tenant isolation

Quotas account for in-flight allocations so cross-controller races can't over-allocate. FairScheduler implements max-min weighted fair sharing; no tenant starved when capacity ≥ tenant count.

09 · Platform

Apple Silicon, end to end.

Apple Virtualization.framework

Thin, correct wrapper. Every call documented against the current Apple reference. No legacy HVF paths, no QEMU, no hidden shim layer.

VirtIO devices

GPU, network (NAT / bridged / isolated), block, sound, entropy, file system, socket, console, keyboard, mouse. Audio can be disabled cleanly for headless compute.

APFS sparse disks

Disk grows as the guest writes. Host space reclaimed on snapshot delete. Sparse container checksummed after writes.

Resumable IPSW downloads

HTTP Range + If-Range, 4-attempt exponential backoff, SHA-256 content-addressed cache. No 15 GB re-downloads on network blips.

Clean Architecture

SpookCore is Foundation-only. SpookApplication may import SpookCore + CryptoKit. SpookInfrastructureApple wraps Apple frameworks. Enforced at build time.

No commercial edition, no paid tier, no "enterprise-only" features behind a wall. The whole binary is what the repo ships.
