Roadmap

What's built, what's next, and where we're headed. Spooktacular serves enterprise CI/CD teams, ML researchers, and remote desktop users.

✓ Shipped ○ In progress □ Planned

v0.1 — Foundation Shipped

Core VM Lifecycle

  • ✓ Create VMs from IPSW restore images
  • ✓ Instant APFS copy-on-write cloning with MachineIdentifier regeneration
  • ✓ Start / stop / delete VMs
  • ✓ VM bundle format (config.json + platform artifacts + disk image)
  • ✓ IPSW version compatibility checking (fails fast, not after 13 GB download)
  • ✓ 89 unit tests across 14 test suites

Hardware Configuration

  • ✓ CPU, memory, disk, displays (1-2), network mode
  • ✓ Audio (speaker + microphone), clipboard sharing
  • ✓ Shared folders (VirtIO file system with automount)
  • ✓ MAC address configuration
  • ✓ Auto-resize display (macOS 14+)
  • ✓ NAT, bridged, isolated, host-only networking

CLI (16 commands)

  • ✓ create, start, stop, list, clone, delete, ip
  • ✓ get, set, snapshot, restore, snapshots
  • ✓ share (add/remove/list), ssh, exec
  • ✓ Styled ANSI terminal output (respects NO_COLOR)
  • ✓ Comprehensive --help with examples on every command

SwiftUI GUI

  • ✓ NavigationSplitView with searchable sidebar
  • ✓ Two-column Create VM sheet with progressive disclosure
  • ✓ Inspector panel for VM configuration
  • ✓ VZVirtualMachineView display with auto-resize
  • ✓ Menu bar extra for quick VM management
  • ✓ Image library in sidebar
  • ✓ Liquid Glass on macOS 26+
  • ✓ Full accessibility (VoiceOver, WCAG 2.1, reduced motion)

Distribution

  • ✓ Proper .app bundle with Info.plist and app icon
  • ✓ Fastlane (test, build, beta, release, docs, lint)
  • ✓ GitHub Actions CI/CD (GitHub Flow)
  • ✓ DocC documentation catalog with articles
  • ✓ GitHub Pages site

v0.2 — Provisioning Engine Next

User Data Execution

  • □ Disk-inject provisioning (mount guest disk, inject LaunchDaemon)
  • □ SSH provisioning (wait for IP, connect, execute script)
  • □ Guest agent over VirtIO socket (vsock)
  • □ Shared-folder watcher provisioning
  • □ Setup Assistant automation via VZ keyboard input (windowed mode)

Built-in Templates

  • □ --github-runner — configure GitHub Actions runner
  • □ --remote-desktop — enable Screen Sharing, report VNC URL
  • □ --buildkite / --circleci — CI runner templates

Remote Desktop

  • □ Screen Sharing (VNC) auto-enable via provisioning
  • □ VNC URL reported in CLI and GUI
  • □ Research: Apple Remote Desktop integration for 4K 60fps
  • □ Research: hardware-accelerated VNC encoding via VideoToolbox
  • □ Research: HEVC/H.265 streaming for near-4K quality at low bandwidth

v0.3 — OCI Distribution Planned

  • □ OCI Distribution Spec v2 client (push/pull)
  • □ Content-addressed chunked disk layers (64 MB, resumable)
  • □ Support for GHCR, Docker Hub, ECR, ACR
  • □ Publish official base images:
    • spooktacular/macos:15.4 — base macOS, SSH enabled
    • spooktacular/macos-xcode:15.4-16.2 — + Xcode + CLI tools
    • spooktacular/macos-ml:15.4 — + Python, MLX, PyTorch MPS
  • □ Image build pipeline (GitHub Actions on Apple Silicon)

v0.4 — Kubernetes Operator Planned

  • □ MacOSVM CRD — single VM as a K8s resource
  • □ MacOSVMPool CRD — managed ephemeral runner pool
  • □ Swift K8s client (SwiftNIO)
  • □ Capacity-aware scheduler (≤2 VMs per Mac)
  • □ Helm chart for operator installation
  • □ Cross-compilation to Linux ARM64
  • □ Container image on GHCR

v0.5 — Autoscaling & Fleet Management Planned

Per-Host Pool Autoscaling

  • □ Runner pool with auto-replacement on exit
  • □ GitHub webhook receiver (workflow_job events)
  • □ GitHub API polling (fallback)
  • □ Configurable min/max replicas and idle timeout

Fleet Scaling (EC2 Mac)

  • □ AWS ASG integration (launch template + user-data)
  • □ Kubernetes cluster-autoscaler integration
  • □ Built-in fleet controller (spook fleet)
  • □ Terraform module (spooktacular/ec2-mac-runners/aws)
  • □ CloudFormation template
  • □ EC2 one-liner setup script

v1.0 — Enterprise Production Planned

Helper Daemon Architecture

  • □ LaunchAgent (desktop) / LaunchDaemon (server) helper
  • □ HTTP control API on configurable interface
  • □ Bearer token authentication
  • □ Server-Sent Events for real-time state streaming
  • □ GUI and CLI as thin HTTP clients to the daemon

Remote Desktop (Production Quality)

  • □ Target: 4K resolution at 60 fps over LAN
  • □ Hardware-accelerated encoding via VideoToolbox (H.265/HEVC)
  • □ Adaptive bitrate based on network conditions
  • □ Low-latency audio sync
  • □ Multi-monitor support (2 displays per VM)
  • □ Clipboard sync between host and remote client
  • □ USB device forwarding research

ML Workload Optimization

  • □ Metal GPU performance profiling and tuning
  • □ Neural Engine access investigation
  • □ Shared memory optimization for large models
  • □ MLX / PyTorch MPS benchmarks vs. native
  • □ Training checkpoint save/restore via VM snapshots
  • □ Federated learning across VM pool (host-only networking)

Mac App Store + Homebrew

  • □ Mac App Store submission (App Sandbox + virtualization entitlement)
  • □ Homebrew Cask formula with CLI symlink
  • □ Developer ID signing + notarization for Homebrew
  • □ Separate MAS and Developer ID entitlements

Future Exploring

  • □ MDM integration — optional embedded Swift MDM server for zero-agent management
  • □ BYO MDM — enroll VMs in customer's Jamf/Kandji/Mosyle
  • □ Web dashboard — browser-based VM management
  • □ REST API documentation (OpenAPI spec)
  • □ Prometheus metrics endpoint for fleet monitoring
  • □ Grafana dashboard templates
  • □ Multi-architecture OCI images (for future Intel support if Apple adds it)
  • □ VM migration between hosts
  • □ Nested virtualization on M3+ (run VMs inside VMs)
