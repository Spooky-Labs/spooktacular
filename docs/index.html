Built for four audiences

Ship the workloads Apple Silicon was meant for.

You already understand your problem. Pick the pillar that matches your team and see the concrete feature set underneath it.

CI/CD

Ephemeral runner pools, at scale.

GitHub Actions, Jenkins, and custom runners on clean macOS VMs. Warm pools, scale-to-zero, drain-before-delete, and a 10-check scrub validator that proves the guest is clean before reuse. Keychain-gated registration tokens — nothing in ps, nothing on disk.

GitHub Actions Jenkins Ephemeral Warm pool
Computer-using agents

A macOS desktop per agent.

Sandbox an agent inside a real Mac — Finder, Safari, Xcode, the lot. A VirtIO guest agent exposes clipboard, exec, app control, filesystem, and listening ports over three scope-separated vsock channels: read-only, runner, and break-glass.

VirtIO vsock Guest agent Scope-separated
Fleet provisioning

Re-image the fleet before standup.

APFS copy-on-write clones a 100 GB golden image in milliseconds. Snapshot / restore per VM. User-data provisioning via disk-inject, SSH, or VirtIO socket. Zero network required.

Virtual Desktop Infrastructure

Persistent Macs for remote developers.

Metal-accelerated VM display with auto-resize, audio, clipboard, shared folders. Built-in Remote Desktop template enables Screen Sharing on first boot. Workspace windows — every VM is an app on your Dock.

Control plane

GUI, CLI, REST API, Kubernetes operator — same primitives.

The SwiftUI app, the spook CLI, the mTLS HTTP server, and the MacOSVM + RunnerPool Kubernetes CRDs all drive the same domain model. Ship a laptop-first demo, then promote to a production cluster without rewriting a thing.

Security posture

Enterprise-grade, from the first commit.

Spooktacular was built for operators who have to pass a Fortune-20 vendor review before the app ships. Every control is documented, tested, and citable in an audit.

  • mTLS everywhere — Keychain-verified cert/key pairs
  • OIDC + SAML with JWKS pinning, strict aud/iss/replay
  • Multi-tenant RBAC with an audit record per decision
  • Break-glass tickets: Secure-Enclave-bound, single-use
  • RFC 6962 Merkle audit tree with Ed25519 tree heads
  • Append-only JSONL + S3 Object Lock dual sinks
  • Per-VM network egress compiled to pf(8) rules
  • Data-at-rest: FileProtectionType bundles
