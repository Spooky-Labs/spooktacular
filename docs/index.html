Why only 2 VMs?

Apple's macOS EULA (Section 2B(iii)) permits up to two additional copies or instances of macOS in virtual environments on each Apple-branded computer. This is also enforced at the kernel level on Apple Silicon. Spooktacular maximizes what Apple allows.

Per Apple's macOS Sequoia SLA Section 2B(iii): "install, use and run up to two (2) additional copies or instances of the Apple Software... within virtual operating system environments on each Apple-branded computer." Permitted for software development, testing, macOS Server, and personal use. Spooktacular enforces this limit in CapacityCheck.swift.

Create your first VM

Install, create, clone, run. No configuration required.

Terminal — zero to running in 60 seconds
# Install
$ brew install --cask spooktacular
 
# Create your first VM from the latest macOS IPSW
$ spook create dev --from-ipsw latest
✓ VM 'dev' created (download + install + setup automated)
 
# Clone for CI (48ms, APFS copy-on-write)
$ spook clone dev runner-01
$ spook clone dev runner-02
 
# Start runners for your pipeline
$ spook start runner-01 --headless --provision ssh \
--template github-runner --var repo=org/app
$ spook start runner-02 --headless --ephemeral \
--provision disk-inject --template github-runner
 
# Check status via the HTTP API
$ curl http://localhost:8080/vms | jq
[{"name":"runner-01","state":"running"},{"name":"runner-02","state":"running"}]
 
# Snapshot before a risky update
$ spook snapshot save runner-01 pre-xcode16
✓ Snapshot 'pre-xcode16' saved
