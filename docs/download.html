Install Spooktacular.

Four install surfaces. Pick the one that matches how you deploy the rest of your stack. The binary is the same; only the packaging differs.

Recommended

Mac App Store

The signed, notarized, sandboxed build. Auto-updates. Free. Includes the bundled spook CLI at /Applications/Spooktacular.app/Contents/MacOS/spook.

Open in App Store
CLI only

Homebrew

For headless hosts, CI, and servers that don't run the GUI. Same binary as the one inside the app bundle.

$ brew tap spooky-labs/tap
$ brew install spook
$ spook doctor
✓ Apple Silicon · Virtualization.framework · 0/2 VMs
Fleet

Helm chart

For Kubernetes-managed Mac fleets. Ships with MacOSVM + RunnerPool CRDs, validating webhook, NetworkPolicy, PDB, ServiceMonitor, and default PrometheusRule.

$ helm repo add spooky-labs \
    https://charts.spooktacular.app
$ helm install spooktacular \
    spooky-labs/spooktacular \
    -n spooktacular-system \
    --create-namespace
Infra-as-code

Terraform module

Provisions dedicated Mac hosts with SSM document, License Manager config, Host Resource Group, 7 CloudWatch alarms, and an ASG lifecycle hook that triggers a graceful drain via the DrainSpooktacularHost runbook. 

module "spooktacular" {
  source  = "spooky-labs/spooktacular/aws"
  version = "~> 1.0"

  hosts = 8
  region = "us-east-1"
}
Same binary, different packaging. The spook shipped in the Homebrew formula, the App Store bundle, and the container image in the Helm chart all come from the same swift build -c release. Reproducible. SLSA provenance attached to every GitHub release.
90 seconds to first boot

Verify → Create → Start.

The installer leaves you with everything you need. No extra dependencies, no sudo steps, no manual Keychain setup for the local case.

first-run.sh
# 1 · Verify the host
$ spook doctor
✓ Apple Silicon (arm64)
✓ macOS 15.4 (min: 14.0)
✓ Virtualization.framework available
✓ Disk: 412 GB free (min: 20 GB)
✓ Capacity: 0/2 VMs running
 
# 2 · Create the first VM (latest macOS IPSW, auto-install)
$ spook create my-mac
Fetching restore image from Apple…
Downloading IPSW (18 GB) … resumable
Installing macOS …
Automating Setup Assistant …
✓ VM 'my-mac' ready.
 
# 3 · Start it
$ spook start my-mac
✓ Running. SSH at 192.168.64.7
Requirements

Apple Silicon only.

Intel Macs can't run macOS in a VM without licensing gymnastics — Apple's Virtualization.framework is Apple-Silicon-only for macOS guests.

Host

Mac with Apple Silicon (M1 / M2 / M3 / M4 / later). macOS 14 (Sonoma) or newer. 16 GB RAM minimum, 32 GB recommended for 2 concurrent VMs.

Storage

20 GB free minimum to bootstrap. APFS-formatted (default). Plan 64 GB+ per VM for a typical dev workload; sparse disks reclaim on delete.

Network

Outbound HTTPS to Apple's signing service for IPSW downloads. Or skip the download with a local IPSW — spook create --from-ipsw <path>.
